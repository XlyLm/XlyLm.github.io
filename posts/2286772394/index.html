二、学习

1、变量

less使用@标识变量

2、混合

混合（Mixin）是一种将一组属性从一个规则集包含（或混入）到另一个规则集的方法。

在混合名后加入()进行混合的使用，注意，你也可以使用 #ids 作为 mixin 使用。

3、嵌套

Less 提供了使用嵌套（nesting）代替层叠或与层叠结合使用的能力。

你还可以使用此方法将伪选择器（pseudo-selectors）与混合（mixins）一同使用。

4、@规则嵌套和冒泡

@ 规则可以与选择器以相同的方式进行嵌套。@ 规则会被放在前面，同一规则集中的其它元素的相对顺序保持不变。

5、运算

算术运算符 +、-、*、/ 可以对任何数字、颜色或变量进行运算。算术运算符在加、减或比较之前会进行单位换算。计算的结果以最左侧操作数的单位类型为准。如果单位换算无效或失去意义，则忽略单位。

乘法和除法不作转换。因为这两种运算在大多数情况下都没有意义

6、calc()特列

为了与 CSS 保持兼容，calc() 并不对数学表达式进行计算，但是在嵌套函数中会计算变量和数学公式的值。

7、转义

转义（Escaping）允许你使用任意字符串作为属性或变量值。任何 ~"anything" 或 ~'anything' 形式的内容都将按原样输出

注意，从 Less 3.5 开始，可以简写为：

8、函数

Less 内置了多种函数用于转换颜色、处理字符串、算术运算等。

9、命名空间和访问符

出于组织结构或仅仅是为了提供一些封装的目的，你希望对混合（mixins）进行分组。

注意：如果不希望它们出现在输出的 CSS 中，例如 #bundle .tab，请将 () 附加到命名空间（例如 #bundle()）后面。

10、映射

从 Less 3.5 版本开始，你还可以将混合（mixins）和规则集（rulesets）作为一组值的映射（map）使用。

11、作用域

Less 中的作用域与 CSS 中的作用域非常类似。首先在本地查找变量和混合（mixins），如果找不到，则从“父”级作用域继承。

与 CSS 自定义属性一样，混合（mixin）和变量的定义不必在引用之前事先定义。

12、注释

块注释和行注释都可以使用

13、导入

你可以导入一个 .less 文件，此文件中的所有变量就可以全部使用了。如果导入的文件是 .less 扩展名，则可以将扩展名省略掉

五、进阶指南

1、变量

变量通过为您提供一种从单个位置控制这些值的方法，使您的代码更易于维护

  • 变量插值

​ 选择器

​ 网址

请注意，在 v2.0.0 之前，仅考虑已在根或当前范围中声明的变量，并且在查找变量时仅考虑当前文件和调用文件。

​ 属性

  • 变量的变量

在 Less 中，您可以使用另一个变量来定义变量的名称。

  • 状态提升

变量不必在使用前声明。

当定义一个变量两次时，使用变量的最后一个定义，从当前范围向上搜索。

  • 作为变量的属性

$prop您可以使用语法轻松地将属性视为变量。

注意，与变量一样，Less 将选择当前/父范围内的最后一个属性作为“最终”值。

  • 默认变量

我们有时会收到对默认变量的请求 - 只有在尚未设置变量时才能设置变量。此功能不是必需的，因为您可以通过在之后放置定义来轻松覆盖变量。

2、父选择器

&运算符表示嵌套规则的父选择器，最常用于将修改类或伪类应用于现有选择器

另一个典型的用法&是产生重复的类名

  • 多&

  • 更改选择器顺序

在继承的（父）选择器之前添加一个选择器会很有用。这可以通过放置&after current 选择器来完成。

  • 组合

&也可用于在逗号分隔列表中生成选择器的所有可能排列

3、扩展

Extend 是一个 Less 伪类，它将它所放置的选择器与匹配它所引用的选择器合并。

  • 扩展语法

扩展要么附加到选择器，要么放入规则集中。它看起来像一个带有选择器参数的伪类，可选地后跟关键字all

它可以包含一个或多个要扩展的类，用逗号分隔。

  • 扩展附加到选择器

附加到选择器的扩展看起来像一个以选择器为参数的普通伪类。一个选择器可以包含多个扩展子句，但所有扩展都必须位于选择器的末尾。

  • 在选择器之后扩展：pre:hover:extend(div pre).
  • 选择器和扩展之间的空格是允许的：pre:hover :extend(div pre).
  • 允许多个扩展：pre:hover:extend(div pre):extend(.bucket tr)- 注意这与pre:hover:extend(div pre, .bucket tr)
  • 这是不允许的：pre:hover:extend(div pre).nth-child(odd). 扩展必须在最后。

如果一个规则集包含多个选择器，它们中的任何一个都可以有 extend 关键字。在一个规则集中扩展的多个选择器

  • 扩展内部规则集

&:extend(selector)可以使用语法将扩展放入规则集的主体中。将extend 放入主体中是一种将其放入该规则集的每个选择器的快捷方式。

  • 扩展嵌套选择器

Extend 能够匹配嵌套的选择器。

  • 与extend精确匹配

默认情况下，扩展查找选择器之间的完全匹配。选择器是否使用前导星并不重要。两个第 n 个表达式具有相同的含义并不重要，它们必须具有相同的形式才能匹配。唯一的例外是属性选择器中的引号，很少有人知道它们具有相同的含义并匹配它们。

  • 第n个表达式

第 N 种表达形式很重要。第 N 个表达式1n+3和n+3是等价的，但 extend 不会匹配它们

  • 扩展全部

当您在扩展参数中指定 all 关键字 last 时，它会告诉 Less 将该选择器作为另一个选择器的一部分进行匹配。选择器将被复制，并且只有选择器的匹配部分将被扩展替换，从而形成一个新的选择器。

  • 带扩展的选择器插值

Extend无法将选择器与变量匹配。如果选择器包含变量，扩展将忽略它。但是，extend 可以附加到插值选择器。

带有变量的选择器将不匹配；

并在目标选择器中使用变量扩展不匹配。

  • 范围/扩展内部@media

目前，:extend内部@media声明只会匹配同一媒体声明中的选择器
