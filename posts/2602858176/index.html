Css预处理器(Sass学习)

  • 官网地址

  • 介绍

Sass 是一门高于 CSS 的元语言，它能用来清晰地、结构化地描述文件样式，有着比普通 CSS 更加强大的功能。Sass 能够提供更简洁、更优雅的语法，同时提供多种功能来创建可维护和管理的样式表。Sass 是采用 Ruby 语言编写的一款 CSS 预处理语言。

Scss 是 Sass 3 引入新的语法，是Sassy CSS的简写，是CSS3语法的超集。

  • 两者的区别

Sass 和 SCSS 其实是同一种东西，我们平时都称之为 Sass，两者之间不同之处有以下两点：

  • 文件扩展名不同，Sass 是以“.sass”后缀为扩展名，而 SCSS 是以“.scss”后缀为扩展名
  • 语法书写方式不同，Sass 是以严格的缩进式语法规则来书写，不带大括号({})和分号(;)，而 SCSS 的语法书写和我们的 CSS 语法书写方式非常类似。

一、安装sass

通过命令行安装 Sass 后，你就可以执行 sass 命令把 .sass 和 .scss 文件编译为 .css 文件。

根据你的操作系统 从 GitHub 上下载对应的安装包并 将可执行文件的路径添加到 PATH 环境变量中。没有外部依赖， 也不需要安装任何其它东西。

二、学习sass

  • sass使用

使用sass 命令将 Sass 编译为 CSS 。 首先你需要告诉 Sass 从哪个文件开始构建，以及将生成的 CSS 输出到哪里。

你还可以利用 --watch 参数来监视单个文件或目录。 --watch 参数告诉 Sass 监听源文件的变化， 并在每次保存 Sass 文件时重新编译为 CSS。

可以使用文件夹路径作为输入和输出， 并使用冒号分隔它们

1.变量

变量是存储信息并在将来重复利用的一种方式，在整个样式表中都可访问。 你可以在变量中存储颜色、字体 或任何 CSS 值，并在将来重复利用。Sass 使用 $ 符号 作为变量的标志。

2.嵌套

Sass 允许您嵌套 CSS 选择器，嵌套方式 与 HTML 的视觉层次结构相同。但不建议使用过度的嵌套规则。

3.片段

您可以创建包含CSS片段的部分Sass文件，这些CSS片段可以包含在其他Sass文件中。这是一个很好的方式来模块化你的CSS，并帮助保持事情更容易维护。片段是一个用前导下划线命名的Sass文件。您可以将其命名为_partial.scss。下划线让Sass知道该文件只是部分文件，不应将其生成为CSS文件。Sass部分与@use规则一起使用。

4.模块(Module)

您不必在单个文件中编写所有的sas。您可以使用@use规则将其按需拆分。此规则将另一个Sass文件作为模块加载，这意味着您可以使用基于文件名的命名空间在Sass文件中引用其变量、混合和函数。使用文件还将在编译输出中包含它生成的CSS！

5.混合(Mixins)

mixin允许您创建一组CSS声明，以便在整个站点中重用。它有助于保持你的页面整洁。您甚至可以传递值，使您的混合更灵活。

使用@mixin指令定义混合并命名，可以为其设置参数；使用@include将混合用作css申明。

6.扩展/继承(Extend/Inheritance)

使用@extend可以在一个选择器和另一个选择器之间共享一组CSS属性。

除了Sass中的占位符类之外，您还可以扩展最简单的CSS选择器，但使用占位符是确保您没有扩展嵌套在样式中其他地方的类的最简单方法，这可能会导致CSS中出现意外选择器。

7.操作符

Sass 支持部分标准的 数学运算符，例如 +、-、*、math.div() 和 %。

三、内容

1.语法

Sass 支持两种不同的语法。两种语法可以互相加载：.sass和.scss

  • scss

和html、css书写解构类似，所有有效的 CSS 也同样都是有效的 SCSS

  • sass

缩进语法是 Sass 的原始语法，因此它使用文件 扩展名 .sass 。 缩进语法支持与 SCSS 相同的所有特性，但是它使用 缩进而不是花括号和分号来描述文档的格式。

Sass 样式表由一系列语句组成，这些语句被评估以构建生成的CSS。一些语句可能有块，使用{and}定义，其中包含其他语句。

这些类型的语句可以在 Sass 样式表的任何地方使用

  • The @error, @warn, and @debug rules.

​ CSS语句

这些语句只能在样式表的顶层使用，或者嵌套在顶层的CSS语句中

  • 导入，使用 @import.

  • 该@extend规则只能在样式规则中使用。
  • 表达式

表达式是位于属性或变量声明右侧的任何内容。每个表达式产生一个*值*。任何有效的CSS属性值也是一个 SASS 表达式。

它们作为参数传递给mixins 和函数，用于使用@if规则进行控制流，并使用算术进行操作。我们将 Sass 的表达式语法称为SassScript。

4.注释

  • scss

SCSS中的注释与 JavaScript 等其他语言中的注释类似。

单行注释以 开头//，一直到行尾。单行注释中的任何内容都不会作为CSS发出，及不会出现在css文件中。它们也被称为静默注释

多行注释：/*…*/，如果多行注释写在允许声明的地方，它会被编译为CSS注释。他们也别称为：loud comment

默认情况下，多行注释会以压缩模式从编译后的CSS中去除。但是，如果注释以/*! 开头，它将始终包含在CSS 输出中。

  • sass

与SCSS一样，使用// 编写的静默注释永远不会作为CSS发出，但与SCSS不同的是，//开头下方缩进的所有内容也会被注释掉。

以缩进开头的缩进语法注释/*以相同的方式工作，只是它们被编译为CSS。因为注释的扩展是基于缩进的，所以关闭*/是可选的。也像SCSS一样， /*注释可以包含插值并且可以开始/*!以避免在压缩模式下被剥离。

  • Documentation Comments

使用 Sass 编写样式库时，您可以使用注释来记录您的库提供的mixins、函数、变量和占位符选择器 ，以及库本身。这些是由SassDoc工具读取的注释，它使用它们来生成漂亮的文档。

文档注释是无声注释，///在您正在记录的内容正上方用三个斜线 ( ) 书写。SassDoc 将注释中的文本解析为Markdown，并支持许多有用的注释来详细描述它。

5.特殊函数

CSS 定义了许多函数，其中大多数都可以与 Sass 的正常函数语法一起正常工作。它们被解析为函数调用，解析为纯 CSS 函数，并按原样编译为 CSS。不过，也有一些例外，它们具有不能仅仅被解析为SassScript 表达式的特殊语法。所有特殊函数调用都返回不带引号的字符串。

  • url()

它可以采用带引号或不带引号的URL。

如果url()参数是一个有效的不带引号的URL，Sass 会按原样解析它。如果它包含变量或函数调用——它会被解析为一个普通的纯CSS函数调用。

  • element(), progid:…(), expression()

6.样式规则

样式规则是 Sass 的基础，您可以使用选择器选择要设置样式的元素，并声明影响这些元素外观的属性。

  • 嵌套

无需一遍又一遍地重复相同的选择器，您可以将一种样式规则写入另一个样式规则中。Sass 会自动将外部规则的选择器与内部规则的选择器结合起来。注意，不要使用深层嵌套

  • 选择器列表

嵌套规则在处理选择器列表（即逗号分隔的选择器）方面很聪明。每个复杂选择器（逗号之间的选择器）都单独嵌套，然后将它们组合回一个选择器列表。

  • 选择器组合器

您也可以嵌套使用组合器的选择器。您可以将组合器放在外部选择器的末尾，内部选择器的开头，或者甚至都放在两者之间。

  • 高级嵌套

如果您想对嵌套样式规则做更多的事情，而不仅仅是将它们按顺序组合并用后代组合器（即纯空格）分隔它们，Sass 可以为您提供支持。有关更多详细信息，请参阅父选择器文档。

  • 插值

您可以使用插值将 变量和函数调用等表达式中的值注入选择器。

Sass 仅在解析插值后解析选择器。这意味着您可以安全地使用插值来生成选择器的任何部分，而不必担心它不会解析。

您可以将插值与父选择器&、@at-root规则和选择器函数结合起来，以便在动态生成选择器时发挥强大的作用。

7.属性申明

在 Sass 中和在 CSS 中一样，属性声明定义了匹配选择器的元素的样式。声明的值可以是任何SassScript 表达式，它将被计算并包含在结果中。

  • 插值

属性的名称可以包含插值，这使得可以根据需要动态生成属性。您甚至可以插入整个属性名称！

  • 嵌套

Sass 通过允许嵌套属性声明，这使属性变得更容易且更少冗余。外部属性名称被添加到内部，用连字符分隔。

其中一些CSS属性具有使用命名空间作为属性名称的简写版本。

  • 隐藏申明

有时您只希望在某些时候出现属性声明。如果一个声明的值是null或一个空的未加引号的字符串，Sass 根本不会将该声明编译成CSS。

  • 自定义属性

CSS自定义属性，也称为CSS变量，具有不寻常的声明语法：它们允许在其声明值中包含几乎任何文本。更重要的是，这些值可供 JavaScript 访问，因此任何值都可能与用户相关。这包括通常会被解析为 SassScript 的值。

不幸的是，插值会从字符串中删除引号，这使得当它们来自 Sass 变量时，很难将带引号的字符串用作自定义属性的值。作为一种解决方法，您可以使用该meta.inspect()函数来保留引号。

8.父选择器

父选择器&是 Sass 发明的一种特殊选择器，在嵌套选择器中用于引用外部选择器。它可以以更复杂的方式重用外部选择器。

当在内部选择器中使用父选择器时，它会被相应的外部选择器替换。

因为父选择器可以被类型选择器替换h1，所以它只允许在复合选择器的开头，其中也允许类型选择器。例如，span&不允许。

  • 添加后缀

您还可以使用父选择器向外部选择器添加额外的后缀。这在使用像BEM 这样使用高度结构化类名的方法时特别有用。只要外部选择器以字母数字名称结尾，您就可以使用父选择器附加附加文本。

  • 在SassScript中

  • 高级嵌套

您可以将&其用作普通的 SassScript 表达式，这意味着您可以将其传递给函数或将其包含在插值中——甚至在其他选择器中！将它与选择器函数和@at-root规则结合使用可以让您以非常强大的方式嵌套选择器。

注意：当 Sass 嵌套选择器时，它不知道使用什么插值来生成它们。这意味着即使您用作&SassScript 表达式，它也会自动将外部选择器添加到内部选择器。这就是为什么你需要明确地使用@at-root规则来告诉 Sass 不要包含外部选择器。

9.占位符选择器

Sass 有一种特殊的选择器，称为“占位符”。它看起来和行为很像一个类选择器，但它以%开头，并且不包含在 CSS 输出中。

与类选择器不同，占位符在没有扩展的情况下不会弄乱CSS，并且它们不要求库的用户在其 HTML 中使用特定的类名。

占位符选择器在编写每个样式规则都可能使用或不使用的 Sass 库时很有用。根据经验，如果您只是为自己的应用程序编写样式表，通常最好只扩展一个可用的类选择器。

10.变量

Sass 变量很简单：给以 开头的名称赋值$，然后可以引用该名称而不是值本身。

变量可以在您想要的任何地方声明。要使用变量，只需将其包含在值中。

CSS有自己的变量，与 Sass 变量完全不同。了解差异！

  • Sass 变量都被 Sass 编译掉了。CSS变量包含在CSS 输出中。
  • CSS变量对于不同的元素可以有不同的值，但 Sass 变量一次只有一个值。
  • Sass 变量是命令式的，这意味着如果你使用一个变量然后改变它的值，之前的使用将保持不变。CSS变量是声明性的，这意味着如果您更改值，它将影响早期使用和后期使用。

Sass 变量与所有 Sass 标识符一样，将连字符和下划线视为相同。这意味着$font-sizeand$font_size都指的是同一个变量。

  • 默认值

通常，当您为变量赋值时，如果该变量已经有一个值，它的旧值将被覆盖。

Sass 提供了!default标志。仅当该变量未定义或其值为null. 否则，将使用现有值。

@use!default加载模块时可以配置定义的变量。

  • 内置变量

不能修改由内置模块定义的变量。

  • 范围

在样式表顶层声明的变量是全局的。这意味着在声明它们之后，可以在其模块中的任何地方访问它们。但并非所有变量都是如此。

  • 流控范围

  • 高级变量函数

12.规则

13.Values

Sass 支持许多值类型，其中大部分直接来自 CSS。每个表达式产生一个值，变量保存值。

  • Numbers, 可能有也可能没有单位，比如12 or 100px。
  • Strings, 可能有也可能没有引号，比如"Helvetica Neue"or bold。
  • Colors, 可以通过其十六进制表示或名称来引用，例如#c6538cor blue，或者从函数返回，例如rgb(107, 113, 127)or hsl(210, 100%, 20%)。
  • Lists of values, 可以用空格或逗号分隔，可以用方括号括起来，也可以根本没有括号，如1.5em 1em 0 2em,Helvetica, Arial, sans-serif或 [col1-start]。

还有一些是特定于 Sass 的：

  • 布尔值true和 false。
  • 单例null 值。
  • Maps 将值与键相关联的映射，例如("background": red, "foreground": pink).
  • 由 .Function references 返回get-function() 和调用的函数引用call()。

14.Operators

Sass 支持一些operators对处理不同值有用的方法。这些包括标准的数学运算符，如+and *，以及各种其他类型的运算符：

  • ==并!=用于检查两个值是否相同。
  • +, -, *, /, 和%对数字具有通常的数学含义，具有与科学数学中的单位使用相匹配的单位的特殊行为。
  • <, <=, >, 并>=检查两个数是否大于或小于另一个数。
  • and, or, 并not具有通常的布尔行为。false Sass 认为除和之外的每个值都是“真”的null。
  • +, -, 和/可用于连接字符串。

  • 单等于

15.内置模块

Sass 提供了许多内置模块，其中包含有用的功能。这些模块可以像任何用户定义的样式表一样使用@use规则加载，并且可以像任何其他模块成员一样调用它们的函数。所有内置模块 sass:都以开头，表明它们是 Sass 本身的一部分。

Sass 提供了以下内置模块：

  • 该sass:math模块提供了对数字进行操作的函数。
  • 该sass:string模块使组合、搜索或拆分字符串变得容易。
  • 该sass:color模块基于现有颜色生成新颜色，从而轻松构建颜色主题。
  • 该sass:list模块允许您访问和修改列表中的值。
  • 该sass:map模块可以在map中查找与键关联的值等等。
  • 该sass:selector模块提供对 Sass 强大的选择器引擎的访问。
  • 该sass:meta模块公开了 Sass 内部工作的细节。

17.命令行

  • 一对一模式

一对一模式将单个输入文件 ( input.scss) 编译到单个输出位置 ( output.css)。如果未传递输出位置，则将编译后的CSS 打印到终端。

如果扩展名是.scss ，则输入文件被解析为SCSS，如果扩展名是.sass，则解析为缩进语法，如果扩展名是.css ，则解析为纯CSS。如果它没有这三个扩展之一，或者它来自标准输入，则默认将其解析为SCSS。

  • 多对多模式

多对多模式将一个或多个输入文件编译为一个或多个输出文件。输入与输出用冒号分隔。它还可以将一个目录下的所有 Sass 文件编译成另一个目录下的同名CSS文件。

在编译整个目录时，Sass 将忽略名称以 . 开头的部分文件_。您可以使用 partials 来分离样式表，而无需创建一堆不必要的输出文件。
